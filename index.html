GDB常用指令

toc: true

GDB常用指令

[TOC]

注意编译时加-g选项，生成信息用于调试

调试时利用-tui选项，会生成文本用户界面显示代码，比较方便

设置断点

  • break line_number，缩写为b line_number

    //当提示没有符号表被读取时 [没有加-g标记] ，可用b 源代码文件名：行号

  • b line_number If condition

  • b func

  • b *address

删除断点

  • delete line_number，缩写为d line_number

  • delete，缩写为d

    //删除所有已经设置的断点，在删除前会显示信息确认是否删除

运行

  • run，缩写r

    //开始执行

  • continue，缩写c

    //离开断点，继续执行

  • next，缩写n

    //执行下一条语句

  • step，缩写s

    //执行下一条语句或者进入函数

  • until，缩写n

    //在循环中时，该指令执行到循环结束后的第一行语句后停止

  • finish

    //继续执行直到函数返回

  • quit，缩写q

    //离开gdb调试

查看信息

  • info break，缩写info b

    //显示断点信息，在哪些地方设置了断点

  • info locals

    //显示局部变量及其值

  • watch varname

    //变量的值发生变化时程序会停止运行

  • print varname，缩写 p

    //显示变量

  • ptype varname

    //显示变量类型

  • print sizeof（varname）

    //显示变量size

  • set var varname = new value

    //修改变量的值

  • print （varname = new value）

    //修改变量的值

  • display varname

    //在每个断点或者每执行一步时都打印该值

其他指令

  • backtrack，缩写bt

    //显示运行到此句的函数堆栈情况

  • disassemble main

    //显示反汇编代码

  • layout asm

    //显示反汇编代码
